This is the annotated source code and documentation for tomafro-deploy, a simple, opinionated set of capistrano deployment recipes. Inspired by this blog post, these recipes use git’s strengths to deploy applications in a faster, simpler manner than a standard capistrano deployment. Using git to manage release versions means apps can be deployed to a single directory. There’s no need for releases, shared or current folders, and no symlinking.
